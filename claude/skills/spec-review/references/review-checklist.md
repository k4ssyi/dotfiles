# 仕様レビュー観点チェックリスト

## 1. 機能要件の抜け漏れ

### 正常系
- 主要なユースケースがすべて定義されているか
- 入力パラメータの型・制約・デフォルト値が明記されているか
- 出力（レスポンス）の構造・型・ステータスコードが定義されているか
- 状態遷移がある場合、遷移図または遷移表が存在するか
- 前提条件（事前条件）と事後条件が明確か

### 異常系
- 各入力パラメータのバリデーションエラー時の挙動が定義されているか
- リソースが存在しない場合の挙動が定義されているか
- 権限不足時の挙動が定義されているか
- 外部サービス障害時のフォールバック・リトライが定義されているか
- タイムアウト時の挙動が定義されているか
- 同時実行・競合状態（Race Condition）への対処が検討されているか

### 境界値
- 数値パラメータの最小値・最大値が定義されているか
- 文字列の長さ制限が定義されているか
- 配列・リストの空/最大件数が定義されているか
- ページネーションの上限・デフォルト値が定義されているか
- 日時に関する境界（深夜0時、タイムゾーン、うるう秒等）が考慮されているか

## 2. 非機能要件の検討漏れ

### パフォーマンス
- 想定リクエスト数（RPS）が見積もられているか
- レスポンスタイムの目標値（P50/P95/P99）が定義されているか
- データ量の増加に対するスケーラビリティが検討されているか
- N+1 クエリやバッチ処理の必要性が検討されているか
- キャッシュ戦略が検討されているか

### セキュリティ
- 認証・認可の要件が明記されているか
- 個人情報・機微情報の取り扱いが定義されているか
- 入力値のサニタイズ要件が検討されているか
- レートリミットの要件が検討されているか
- 監査ログの要件が検討されているか

### 運用
- ログ出力の要件が定義されているか
- モニタリング・アラートの要件が検討されているか
- デプロイ戦略（段階的ロールアウト等）が検討されているか
- ロールバック手順が検討されているか
- データマイグレーションの必要性と手順が検討されているか
- フィーチャーフラグによる段階的リリースが検討されているか

## 3. ユーザーストーリー間の矛盾・依存関係

### 矛盾の検出
- 同じデータに対して異なる仕様が定義されていないか
- 状態遷移が他の機能の前提条件と矛盾していないか
- 用語・名称が文書全体で統一されているか
- 権限モデルが機能間で一貫しているか

### 依存関係
- 機能間の実行順序に暗黙の依存がないか
- 共有リソース（DB テーブル、キュー等）への依存が明示されているか
- 外部サービスへの依存が一覧化されているか
- 他チーム・他サービスとの連携ポイントが明確か
- リリース順序の制約が特定されているか
