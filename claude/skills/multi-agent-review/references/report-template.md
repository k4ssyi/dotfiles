# Multi-Agent Review Report Template

レビュー結果の統合時に使用するMarkdownテンプレート。

## テンプレート

```markdown
# コードレビュー総合結果

## レビュー対象

- **diffソース**: {親ブランチとの差分 (`{base_branch}`) / 現在の変更}
- **レビューエージェント**: {実行したエージェント一覧}

## サマリー

| 観点 | BLOCKER | CRITICAL | MAJOR | MINOR | 却下 |
|------|---------|----------|-------|-------|------|
| アーキテクチャ | 0 | 0 | 0 | 0 | 0 |
| コード品質・型安全性・API設計 | 0 | 0 | 0 | 0 | 0 |
| データベース設計 | 0 | 0 | 0 | 0 | 0 |
| セキュリティ（実行時のみ） | 0 | 0 | 0 | 0 | 0 |

**検証率**: {検証済み件数}/{全指摘件数} ({採用率}%)
**マージ可否:** PASS / FAIL

---

## 重大な問題（BLOCKER / CRITICAL）

### [{severity}] {issue_title} — 信用度: {score}%
- **検出元**: {agent}
- **箇所**: `{file_path}:{line}`
- **内容**: {description}
- **修正案**: {recommendation}
- **検証根拠**: {verification_evidence}

---

## 要改善（MAJOR）

### [MAJOR] {issue_title} — 信用度: {score}%
- **検出元**: {agent}
- **箇所**: `{file_path}:{line}`
- **内容**: {description}
- **修正案**: {recommendation}
- **検証根拠**: {verification_evidence}

---

## 改善推奨（MINOR）

### [MINOR] {issue_title} — 信用度: {score}%
- **検出元**: {agent}
- **箇所**: `{file_path}:{line}`
- **検証根拠**: {verification_evidence}

---

## 良い点

| 観点 | 内容 |
|------|------|
| {agent} | {positive_point} |

---

## 確認依頼事項

- [ ] {action_item}

---

## 却下した指摘

| 指摘 | 重要度 | 検出元 | 却下理由 |
|------|--------|--------|---------|
| {issue_title} | {severity} | {agent} | {rejection_reason} |
```

## 重要度の定義

| レベル | 意味 | マージ可否 |
|--------|------|-----------|
| **BLOCKER** | マージ不可。即座に修正必須 | NG |
| **CRITICAL** | 重大な問題。リリース前に修正必須 | 要判断 |
| **MAJOR** | 修正すべき問題 | 修正後マージ可 |
| **MINOR** | 改善推奨 | マージ可 |

## 信用度スコアの読み方

| スコア | 意味 |
|--------|------|
| **90〜100%** | コード・ルール両方で裏付け済み。対応推奨 |
| **65〜89%** | コードまたはルールで確認済み。確認の上対応 |
| **40〜64%** | 要判断。コンテキスト依存の可能性あり |
| **0〜39%** | 却下（却下リストに理由記載） |

## ルール

- 問題がないセクションは省略可
- 複数エージェントが同じ箇所を指摘した場合、検出元を併記して重複を排除する
- 良い点セクションは必ず含める（レビューのバランスを保つため）
- 条件付きエージェント（database-architect, security-auditor）が未実行の場合、対応するサマリー行を省略
- 採用された指摘のみメインセクションに記載し、却下分は末尾の却下リストにまとめる
- サマリーテーブルの件数は**採用分のみ**をカウントする（却下列は参考値）
